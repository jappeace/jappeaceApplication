* Summary

* Practical rules
+ containerise your LLM environment
+ force integration tests per feature in CLAUDE.md

+ two questions: do tests check the right thing? is the code sane?
+ formatter in CI for style, decompose tasks for architecture
+ let go of the details --- that's what makes vibing work
